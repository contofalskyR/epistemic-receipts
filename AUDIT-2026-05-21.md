# Epistemic Receipts — Pipeline Audit Report

**Date:** 2026-05-21
**Auditor:** automated read-only inspection
**Scope:** `scripts/`, `prisma/schema.prisma`, production DB, `.env.local` keys

## Summary

- **DB schema is in sync** — `prisma migrate status` reports up to date. No migration drift.
- **5 ingester source endpoints are unreachable or unhealthy** from the audit host: Pakistan, Indonesia, T&T (503), Riksdag (RST on full GET), and EU Parliament adopted-texts (timeout). Other 4xx/403 statuses on root URLs are expected for API endpoints that need params/auth.
- **3 pipeline tags have no corresponding ingester script in `scripts/`**: `retraction_watch_v1` (55 claims, 110 sources, 0 edges), plus the registry-stale `iau_v1` / `solar_system_v1` which are actually covered by `ingest-astronomy.ts`.
- **`uspto_v1` retirement is incomplete on the Edge layer**: 97 Edges remain with `deleted = false` even though all 182 claims are flagged `verificationStatus = DEPRECATED`. The retirement playbook should cascade to edges.
- **Data quality hotspots**: `nih_reporter_v1` (48.95%) and `uspto_v1` (59.41%) Sources are missing `url`; several pipelines (`nz_*`, `pubchem_v1`, `solar_system_v1`) have 100% NULL `publishedAt`.

## Recommended Fixes

### Critical

1. **Investigate `nobel_v1` deprecation: 662 of 1,688 claims (39%) are flagged `verificationStatus = DEPRECATED` but the pipeline is still listed as Active in `AGENTS.md`.** Either the registry is wrong or a partial retirement happened without documentation. Trace which records were deprecated and why; the audit trail principle in `AGENTS.md` requires this be recorded.

2. **Complete the `uspto_v1` retirement on the Edge layer.** 97 Edges still have `deleted = false` even though the Claims they back are DEPRECATED. Public listing views that filter on Edge state (not just Claim state) will still show retired evidence. Either soft-delete the edges or document why they're intentionally retained.

### High

3. **`retraction_watch_v1` has 55 claims and 110 sources but zero edges and no ingester script in the repo.** Either find/restore the script (check git history), move the records to `manual`, or deprecate them. Edges-of-zero means no FOR/AGAINST signal is being expressed by those records.

4. **Indonesia (`peraturan.go.id`) and Pakistan (`pakistancode.gov.pk`) ingesters cannot reach their hosts from this audit host.** Verify whether these ingesters work from the production worker — if not, the pipelines are stalled. Both have non-trivial DB footprints implied by their pipeline registry entries (and dry-run JSON samples exist).

5. **T&T legal-affairs site returns 503.** `ingest-tt.ts` should be marked as stalled or fall through to Wayback like `paclii`/`uae`.

6. **`enrich-vote-counts.ts` Bundestag and Knesset handlers are explicit placeholders** (L411–421, L464–474) — they always return `counts.skipped`. This explains the `LegislativeVote` table only having 879 rows despite 219,965 PoliticalContext rows. Either implement the XML download path documented in the comment or split these out of the enricher so the placeholders don't masquerade as runs.

### Medium

7. **`nih_reporter_v1` and `uspto_v1` Source records: ~49–59% missing `url`.** For an audit-trail platform, a Source without a URL violates the verifiability principle. Backfill from `externalId` or deprecate the unverifiable records.

8. **`Source.publishedAt` is 100% NULL for `nz_*`, `pubchem_v1`, `solar_system_v1`.** Either backfill from canonical sources (PubChem CID has a creation date; NZ Acts have enactment dates) or document that these pipelines intentionally omit `publishedAt`.

9. **Registry table in `AGENTS.md` is stale.** `icd11_v1` (0 → 1,374), `uk_legislation_v1` (0 → 11,777), `nato_official_texts_v1` (0 → 454). Either auto-generate this table or refresh it as part of the pipeline-run checklist.

10. **`SuggestedThresholdEvent` and `SourceCredibilityEvent` tables are empty.** Both are central to the "humans promote suggestions" workflow but have no producer. Either implement the writer or remove the models if the workflow has been replaced.

### Low

11. **Remove `LAWS_AFRICA_TOKEN` from `.env.local`** — referenced nowhere in code.

12. **`scripts/_check_db.ts`** appears to be a stale ad-hoc utility. Move to `scripts/retired/` or delete.

13. **Riksdag base URL returns 200 on HEAD but resets on full GET** from this host. Ingester works via paginated fetches, but if it ever stalls, the WAF is the likely cause — document a `Mozilla/5.0` UA / rate-limit policy.

14. **PaCLII, UAE, Singapore live hosts return 403 on bare GETs.** Ingesters mitigate via Wayback CDX, but live freshness is impossible. Document the Wayback dependency in the per-pipeline header comments so it's not surprising on next maintenance pass.

15. **EU Parliament adopted-texts endpoint times out.** 4,331 records are in DB already; if this is a known dead endpoint, document it.

16. **Prisma major-version bump available** (6.19.3 → 7.8.0). Out of audit scope, but flagged so it doesn't surprise the upgrade window.
