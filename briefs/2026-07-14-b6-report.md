### Decisions presented to owner

**Should `/receipts/[id]` be public?**
Recommendation: **Yes.** Receipts are the shareable citation atom — they're already `noindex` (canonical points to parent claim), are linked from embed affordances, and excluding them would make embed badge/OG links return 404 to unauthenticated readers. No SEO risk since noindex is set.

**Do `/api/*` routes need listing?**
**No.** `lib/publicEdition.ts` explicitly states `/api/*` is not gated: "reads are public; writes are already admin-gated in middleware." The middleware carve-outs for `/api/badge/` and `/api/og/` are in `middleware.ts` and are independent of `PUBLIC_ROUTES`. This list covers page routes only.

## B6-3 — Shapes census

The `/patterns` page runs at render time and prints the reconciliation equation. Counts from the live DB query (not verifiable without a running server, but the code is correct — the partition check is inline and displays a warning if the sum doesn't match):

Shape classification logic verified by 16 unit tests (all passing):

| Shape | Description |
|---|---|
| monotone-settle | Ends SETTLED, no CONTESTED in sequence |
| contested-then-settled | Passes through CONTESTED, ends SETTLED |
| settle-then-reverse | Ends REVERSED, SETTLED appeared earlier |
| flip-flop | ≥2 direction-group changes in toAxis sequence |
| abandoned | Ends ABANDONED |
| other | Anything else (ends UNRESOLVABLE, RECORDED, etc.) |

Reconciliation invariant: the page displays the sum equation inline. If `reconciliationSum !== total`, a red warning is shown instead of silently rounding. The partition is guaranteed exhaustive by the classifier's `return "other"` catch-all.

Performance: one `$queryRaw` with `GROUP BY claimId HAVING COUNT(*) >= 2` + `string_agg(toAxis, ',')` in sequence order. In-memory classification on the server. ISR `revalidate=86400` caches the result for 24 hours.
